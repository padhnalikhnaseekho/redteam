# Spec Gate — Repo Rule

> **Generated by:** `sdd-init` or `sdd-enable` (do not edit by hand — re-run with `--repair` to regenerate).
> **Enforced by:** `claude/hooks/spec-gate.sh` (PreToolUse(Write|Edit), exit code 2 blocks).
> **Authoritative config:** `.claude/SDLC-PROFILE.md` (this file is human/AI-readable documentation; the hook reads env vars set by `session-start.sh` from the profile).

---

## What the gate does

Every `Write` or `Edit` tool call is intercepted by the spec-gate hook before
it runs. The hook decides whether to allow the write based on three things:

1. **What path is being written?**
2. **Which agent is doing the writing (or is it the main session)?**
3. **Does an approved feature file exist?**

The hook follows the Anthropic Claude Code PreToolUse contract:
- exit 0 → allow the tool call
- exit 1 → show error to the user (does not block)
- exit 2 → **block** the tool call and inject the stderr message into the model context

See: https://docs.claude.com/en/docs/claude-code/hooks

## Always-exempt paths (built into the hook)

These paths can be written without an approved feature file:

- `specs/` — spec files are how the gate opens, not what it blocks
- `tests/` — tests are exempt to support TDD inside an in-progress feature
- `docs/` — documentation
- `.claude/agent-memory/` — each agent's persistent memory
- `.claude/SDLC-PROFILE.md` — gate configuration
- `.claude/rules/` — this file and any other rule docs
- `package.json`, `package-lock.json`, `README.md`, `CLAUDE.md`, `GEMINI.md`, `.gitignore`, `.github/`

Additional exempt paths come from the `- exempt-paths:` key in `SDLC-PROFILE.md`.

## Always-blocked paths

- `.git/` — immutable. The hook returns exit 2 with no exception.

## Gate rules

Outside the exempt list, every path goes through these checks:

1. **Main-session direct writes are blocked.** If `agent_type` is empty in the
   hook payload, the gate refuses with a message telling the model to delegate
   via the `Agent` tool (`spec-driven-developer`, `spec-writer`, `feature-planner`).
2. **Subagent writes require an approved feature.** The hook counts feature
   files under `specs/features/active/` that have `status: SPEC-APPROVED` in their YAML
   frontmatter. If at least one exists, the write is allowed and the
   authorising feature file is recorded in stderr as an audit trail.
3. **IN-PROGRESS fallback.** If no `SPEC-APPROVED` file exists but an
   `IN-PROGRESS` one does, the write is also allowed (the spec has already
   moved past approval into active implementation).
4. **No approved or in-progress feature → exit 2.** The hook tells the model
   to invoke `feature-planner` first.

---

## How to open the gate for a new feature

1. Tell the model: *"build feature X"* (or whatever the change is).
2. The model invokes `feature-planner` → writes
   `specs/features/active/<feature-name>.md` with `status: DRAFT`.
3. `spec-writer` reviews and flips status to `SPEC-APPROVED`.
4. `spec-driven-developer` can now Write/Edit code under `src/`.
5. After implementation, `code-review-specialist` reviews and the feature
   moves to `specs/features/completed/`. The gate closes for that feature.

---

## Cross-directory enforcement

When a session is opened at a parent directory above an SDD-enabled repo,
`session-start.sh` won't see `.claude/SDLC-PROFILE.md` at the project root,
so `SDD_ENABLED=false`. To prevent that as a bypass, `spec-gate.sh` walks
up from the target file path until it finds a `.claude/SDLC-PROFILE.md` and
reuses that profile's gate settings. (See ADR-015 in `ARCHITECTURE.md`.)

---

## Disabling the gate

The gate cannot be bypassed from inside a Claude Code session. Per Anthropic:

> A hook returning `permissionDecision: deny` (or, equivalently, exit code 2
> on a `PreToolUse` hook) will block a tool even in bypassPermissions mode or
> with `--dangerously-skip-permissions`.

To turn the gate off for a repo, delete `.claude/SDLC-PROFILE.md` (or set
`SDD_ENABLED=false` in your environment). The hook will then allow all
writes — but agents that depend on the profile will also stop working.
